# KDCA API 상세 요청사항

**문의처:** nalja01@korea.kr / 043-719-7735
**요청자:** Sentinel Korea 호흡기 감염병 감시 시스템 운영팀

---

## 배경

현재 Sentinel Korea는 질병관리청이 매주 발표하는 **엑셀/PDF 보고서**를 수동 다운로드 후 파싱하여 시스템에 반영하고 있습니다. 이 수동 과정을 **API 자동 수집**으로 전환하고자 합니다.

현재 `EIDAPIService/PeriodRegion` API를 통해 법정감염병 전수신고 데이터는 자동 수집 중이나, 아래 4개 감시체계 데이터는 API가 제공되지 않아 매주 엑셀을 수동 업로드하고 있습니다.

---

## 요청 API 1: 급성호흡기감염증 (ARI) 표본감시

### 검색 조건
| 조건 | 값 / 범위 |
|------|-----------|
| 감시체계 | 급성호흡기감염증 표본감시 |
| 기간 유형 | **주차별** (epiweek) |
| 기간 범위 | 시작연도~종료연도 (예: 2024~2025) |
| 지역 | **전국 + 17개 시/도별** (서울, 부산, 대구, 인천, 광주, 대전, 울산, 세종, 경기, 강원, 충북, 충남, 전북, 전남, 경북, 경남, 제주) |

### 필요 응답 항목
| 항목 | 설명 | 예시 |
|------|------|------|
| `period` | 기간 (연도-주차) | "2025년 17주" |
| `region` | 시/도명 | "서울특별시" |
| `total` | 전체 급성호흡기감염증 환자 수 | 1234 |
| `pathogens.마이코플라즈마균` | 병원체별 검출 건수 | 45 |
| `pathogens.클라미디아균` | | 12 |
| `pathogens.아데노바이러스` | | 78 |
| `pathogens.사람보카바이러스` | | 23 |
| `pathogens.파라인플루엔자바이러스` | | 34 |
| `pathogens.호흡기세포융합바이러스` | RSV | 56 |
| `pathogens.리노바이러스` | | 89 |
| `pathogens.사람메타뉴모바이러스` | hMPV | 67 |
| `pathogens.사람코로나바이러스` | seasonal CoV | 11 |
| `pathogens.인플루엔자바이러스` | | 90 |
| `pathogens.코로나19바이러스` | COVID-19 | 45 |

### 현재 수동 작업 방식
- 질병청 웹사이트에서 "급성호흡기감염증 선택됨 - 전체[전체] - 17wk.xlsx" 다운로드
- 엑셀 파싱 후 `kdca_ari_weekly.json`으로 변환

## 요청 API 2: 인플루엔자 의사환자분율 (ILI)

### 검색 조건
| 조건 | 값 / 범위 |
|------|-----------|
| 감시체계 | 인플루엔자 표본감시 |
| 기간 유형 | **주차별** (epiweek, 시즌 36주~다음해 35주) |
| 기간 범위 | 시즌 단위 (예: 2024-2025 시즌) |
| 지역 | **전국 + 17개 시/도별** |

### 필요 응답 항목
| 항목 | 설명 | 예시 |
|------|------|------|
| `period` | 기간 | "2025년 17주" |
| `region` | 시/도명 | "서울특별시" |
| `ili_rate` | 의사환자분율 (ILI/1,000명) | 7.2 |

### 현재 수동 작업 방식
- "인플루엔자 선택됨 - 17wk.xlsx" 다운로드
- 엑셀 파싱 후 `kdca_influenza_ili_weekly.json`으로 변환

## 요청 API 3: 중증급성호흡기감염증 (SARI) 감시

### 검색 조건
| 조건 | 값 / 범위 |
|------|-----------|
| 감시체계 | 중증급성호흡기감염증 입원 감시 |
| 기간 유형 | **주차별** |
| 기간 범위 | 시즌 단위 (36주~35주) |
| 지역 | **전국 + 17개 시/도별** |

### 필요 응답 항목 (폐렴 + 인플루엔자 구분)
| 항목 | 설명 | 예시 |
|------|------|------|
| `period` | 기간 | "2025년 17주" |
| `region` | 시/도명 | "서울특별시" |
| `sari_total` | SARI 전체 입원 환자 수 | 150 |
| `sari_pneumonia_cases` | SARI 중 폐렴 환자 수 | 45 |
| `sari_influenza_cases` | SARI 중 인플루엔자 환자 수 | 23 |
| `case_ratio_pneumonia` | 폐렴 비율 | 0.30 |
| `case_ratio_influenza` | 인플루엔자 비율 | 0.15 |

### 현재 수동 작업 방식
- "중증급성호흡기감염증 선택됨 환자 중 폐렴 환자 현황 - 17wk.xlsx" 다운로드
- "중증급성호흡기감염증 선택됨 환자 중 인플루엔자 환자 현황 - 17wk.xlsx" 다운로드
- 각각 파싱 후 `kdca_sari_pneumonia_weekly.json`, `kdca_sari_influenza_weekly.json`으로 변환

## 요청 API 4: 하수기반 감염병 감시 (폐하수 병원체)

### 검색 조건
| 조건 | 값 / 범위 |
|------|-----------|
| 감시체계 | 하수기반 감염병 감시 |
| 기간 유형 | **주차별** |
| 기간 범위 | 시작연도~종료연도 |
| 지역 | **전국 + 시/도별** (또는 하수처리장별) |

### 필요 응답 항목
| 항목 | 설명 | 예시 |
|------|------|------|
| `period` | 기간 | "2025년 17주" |
| `region` | 시/도명 또는 처리장명 | "서울특별시" |
| `sars_cov2` | SARS-CoV-2 농도 (copies/L) | 12500 |
| `influenza_a` | Influenza A 농도 | 3400 |
| `influenza_b` | Influenza B 농도 | 890 |
| `rsv` | RSV 농도 | 2100 |
| `norovirus` | Norovirus 농도 | 5600 |
| `normalized_index` | 종합 병원체 지수 (정규화) | 0.72 |

### 현재 수동 작업 방식
- "하수기반감염병감시 주간분석보고" PDF를 다운로드
- PDF에서 표/그래프를 수동으로 읽어 시스템 반영 (가장 비효율적)

## 기술 사양 (희망)

| 항목 | 희망 사양 |
|------|-----------|
| 응답 형식 | **JSON** (기존 EIDAPIService와 동일한 `response.body.items` 구조) |
| 인증 방식 | 기존 공공데이터포털 서비스키 (ServiceKey) 방식 |
| 페이징 | `pageNo`, `numOfRows` 파라미터 (기존 동일) |
| 호출 빈도 | 주 1~2회 (매주 화~수 갱신 시점) |
| 데이터 갱신 주기 | 주간 (감시 주간 보고서 발표와 동일) |

### 요청 파라미터 예시 (기존 API 패턴 준용)
```
GET https://apis.data.go.kr/{서비스명}/{엔드포인트}
?serviceKey={인증키}
&resType=2                    (JSON)
&searchPeriodType=3           (주간)
&searchStartYear=2025
&searchEndYear=2025
&searchRegion=11              (서울=11, 부산=21, ... 또는 전체)
&pageNo=1
&numOfRows=1000
```

## 활용 목적

**Sentinel Korea** — 호흡기 감염병 통합 감시 대시보드로, 17개 시/도의 호흡기 감염병 위험도를 실시간 모니터링합니다.

현재 6개 감시 신호를 종합하여 지역별 위험 등급(G0~G3)을 산출합니다:
1. **전수신고 감염병** ← 기존 API로 자동 수집 중
2. **폐하수 병원체 농도** ← PDF 수동 (API 요청)
3. **SARI 폐렴** ← 엑셀 수동 (API 요청)
4. **SARI 인플루엔자** ← 엑셀 수동 (API 요청)
5. **ILI 의사환자분율** ← 엑셀 수동 (API 요청)
6. **급성호흡기감염증 (ARI)** ← 엑셀 수동 (API 요청)

위 4개 API가 제공되면, 매주 수동 엑셀/PDF 업로드 과정이 완전 자동화되어 데이터 갱신 지연 없이 실시간 감시가 가능합니다.

---

## 우선순위

1. **ILI 의사환자분율 (시/도별, 주차별)** — 가장 핵심적인 호흡기 감시 지표
2. **SARI 폐렴 + 인플루엔자 (시/도별, 주차별)** — 중증도 감시의 핵심
3. **급성호흡기감염증 ARI + 병원체별 (시/도별, 주차별)** — 병원체 동향 파악
4. **하수기반 병원체 (시/도별, 주차별)** — 조기경보 신호로 가장 중요하나 데이터 구조가 복잡할 수 있음
