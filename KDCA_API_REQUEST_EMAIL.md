안녕하세요, 담당자님.

API 관련 문의에 답변해 주셔서 감사합니다. 요청드린 내용에 대해 구체적인 검색 조건과 조회 결과 항목을 아래와 같이 정리하였습니다.

---

현재 저희 시스템(Sentinel Korea)은 호흡기 감염병 통합 감시 대시보드로, 17개 시/도의 호흡기 감염병 위험도를 실시간 모니터링하고 있습니다. 현재 질병관리청의 EIDAPIService/PeriodRegion API를 통해 법정감염병 전수신고 데이터는 자동 수집하고 있으나, 아래 4개 감시체계 데이터는 매주 엑셀 파일 또는 PDF를 직접 다운로드하여 수동으로 시스템에 반영하고 있는 상황입니다. 이 수동 과정을 API 자동 수집으로 전환하고자 합니다.


1. 급성호흡기감염증(ARI) 표본감시

검색 조건으로는 주차별(epiweek) 기간 유형에, 시작연도~종료연도 범위 지정이 가능하고, 지역은 전국 합계 및 17개 시/도별로 조회할 수 있으면 좋겠습니다. 응답 항목으로는 기간(연도-주차), 시/도명, 전체 환자 수, 그리고 병원체별 검출 건수(마이코플라즈마균, 클라미디아균, 아데노바이러스, 사람보카바이러스, 파라인플루엔자바이러스, 호흡기세포융합바이러스, 리노바이러스, 사람메타뉴모바이러스, 사람코로나바이러스, 인플루엔자바이러스, 코로나19바이러스)가 필요합니다. 현재는 질병청 웹사이트에서 "급성호흡기감염증" 엑셀 파일을 매주 다운로드하여 파싱하고 있습니다.


2. 인플루엔자 의사환자분율(ILI) 표본감시

검색 조건은 주차별 기간 유형에, 시즌 단위(36주~다음해 35주) 조회가 가능하고, 17개 시/도별로 구분되면 좋겠습니다. 응답 항목으로는 기간, 시/도명, 의사환자분율(ILI/1,000명)이 필요합니다. 현재는 "인플루엔자" 엑셀 파일을 매주 수동 다운로드하여 사용 중입니다.


3. 중증급성호흡기감염증(SARI) 입원 감시

검색 조건은 주차별, 시즌 단위이며, 17개 시/도별 조회를 희망합니다. 응답 항목으로는 기간, 시/도명, SARI 전체 입원 환자 수, 그중 폐렴 환자 수와 인플루엔자 환자 수가 각각 구분되어 제공되면 좋겠습니다. 현재는 "중증급성호흡기감염증 중 폐렴 환자 현황"과 "중증급성호흡기감염증 중 인플루엔자 환자 현황" 엑셀 파일 2종을 매주 따로 다운로드하고 있습니다.


4. 하수기반 감염병 감시(폐하수 병원체)

검색 조건은 주차별이며, 시/도별 또는 하수처리장별로 조회가 가능하면 좋겠습니다. 응답 항목으로는 기간, 지역명, SARS-CoV-2/인플루엔자A/인플루엔자B/RSV/노로바이러스 등 병원체별 농도(copies/L 또는 정규화 지수)가 필요합니다. 현재는 "하수기반감염병감시 주간분석보고" PDF를 다운로드하여 표와 그래프에서 수치를 수동으로 읽어내고 있어 가장 비효율적인 부분입니다.


기술적으로는 기존 EIDAPIService와 동일한 JSON 응답 형식(response.body.items 구조)과 공공데이터포털 서비스키 인증 방식, pageNo/numOfRows 페이징을 희망합니다. 호출 빈도는 주 1~2회 수준이며, 감시 주간 보고서 발표 시점에 맞추어 데이터가 갱신되면 충분합니다.

위 4개 API가 제공되면 현재 매주 수행하는 엑셀/PDF 수동 업로드 과정이 완전히 자동화되어 데이터 갱신 지연 없이 실시간 감시 운영이 가능해집니다. 전부가 어려우시다면 우선순위로는 ILI 의사환자분율 > SARI 폐렴/인플루엔자 > ARI 병원체별 > 하수기반 병원체 순으로 중요도가 높습니다.

추가로 필요한 정보가 있으시면 말씀해 주시기 바랍니다. 감사합니다.
